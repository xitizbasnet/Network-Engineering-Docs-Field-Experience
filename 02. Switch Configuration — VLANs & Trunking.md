# Objective

Configure VLANs on managed switches, set access ports for end devices, configure trunk ports for inter-switch and uplink connections, and verify VLAN isolation.

Before starting, ensure the following requirements are met:

* Cisco IOS switch
  *(or HP ProCurve / Aruba / Ubiquiti equivalent)*

* Console cable + PuTTY
  **OR**

# Part A — Basic Switch Initialisation

## Step 1: Connect to the Switch

🔌 Connect via:

* Console cable using:

  * `9600 baud`
  * `8N1`

**OR**

* SSH to the switch management IP

---

## Step 2: Enter Privileged EXEC Mode

🔐 Enter privileged mode and provide the enable password.

```bash
enable
```

---

## Step 3: Configure Hostname

🖥️ Set the switch hostname.

```bash
hostname SW-ACCESS-01
```

---

## Step 4: Disable Unused Ports

🚫 Shut down unused interfaces for security and network hygiene.

```bash
interface range GigabitEthernet 0/21-24
shutdown
```

---

## Step 5: Configure Management IP Address

🌐 Assign the management IP to the VLAN interface.

```bash
interface Vlan10
ip address 192.168.10.11 255.255.255.0
no shutdown
```

---

## Step 6: Configure Default Gateway

📡 Define the switch default gateway.

```bash
ip default-gateway 192.168.10.1
```

---

## Step 7: Enable SSH Access

🔐 Configure SSH for secure remote administration.

```bash
ip domain-name company.local
crypto key generate rsa modulus 2048
ip ssh version 2
```

---

## Step 8: Save Configuration

💾 Save the running configuration to startup configuration.

```bash
copy running-config startup-config
```

# Part C — Access Port Configuration (End Devices)

## Configure Access Ports

🖧 Assign switch ports to the correct VLANs for end-user and device connectivity.

---

## Example 1 — Staff PC Port

```bash
interface GigabitEthernet 0/1   ! Staff PC
 switchport mode access
 switchport access vlan 20
 spanning-tree portfast
 no shutdown
```

### Configuration Notes

| Setting                     | Purpose                                 |
| --------------------------- | --------------------------------------- |
| `switchport mode access`    | Sets the port as an access port         |
| `switchport access vlan 20` | Assigns the port to Staff VLAN          |
| `spanning-tree portfast`    | Speeds up port transition for endpoints |
| `no shutdown`               | Enables the interface                   |

---

## Example 2 — IP Camera Port

```bash
interface GigabitEthernet 0/10   ! IP Camera
 switchport mode access
 switchport access vlan 40
 spanning-tree portfast
 no shutdown
```

# Part D — Trunk Port Configuration (Uplink to Core)

## Configure Trunk Port

🔀 Configure uplink interfaces to carry multiple VLANs between switches.

```bash
interface GigabitEthernet 0/24   ! Uplink to Core Switch
 switchport mode trunk
 switchport trunk encapsulation dot1q
 switchport trunk allowed vlan 10,20,30,40,50,60
 switchport trunk native vlan 10
 no shutdown
```

---

## Trunk Configuration Details

| Command                                | Purpose                          |
| -------------------------------------- | -------------------------------- |
| `switchport mode trunk`                | Enables trunking                 |
| `switchport trunk encapsulation dot1q` | Uses IEEE 802.1Q tagging         |
| `switchport trunk allowed vlan`        | Restricts VLANs allowed on trunk |
| `switchport trunk native vlan 10`      | Sets native VLAN                 |
| `no shutdown`                          | Enables the interface            |

# Verification Commands

## VLAN & Trunk Verification

Use the following commands to verify proper VLAN and trunk operation.

| Command                      | Purpose                                      |
| ---------------------------- | -------------------------------------------- |
| `show vlan brief`            | List all VLANs and assigned ports            |
| `show interfaces trunk`      | Verify trunk ports and allowed VLANs         |
| `show spanning-tree vlan 40` | Check STP root and port states for CCTV VLAN |
| `show ip interface brief`    | Verify management IP is operational          |
| `ping 192.168.10.1`          | Test gateway reachability from switch        |

# Best Practice Tips

## 🔒 VLAN Security

* Always specify `switchport trunk allowed vlan` explicitly.
* Never use `all` in production environments.
* Allowing all VLANs can lead to unnecessary VLAN flooding across trunks.

---

## 🛡️ Native VLAN Security

* Set the native VLAN to an unused VLAN.
* Example:

  * `VLAN 10`
  * `VLAN 999`

This helps prevent:

* VLAN hopping attacks
* Misconfigured trunk exploitation

---

## ⚡ PortFast Usage

* Enable `PortFast` only on access ports connected to end devices.
* **Never enable PortFast on trunk ports.**

Improper use may cause:

* Spanning Tree instability
* Network loops

---

## 🚨 BPDU Guard Protection

Enable BPDU Guard on access ports to prevent rogue switches from connecting.

```bash
spanning-tree bpduguard enable
```

### Benefits

* Prevents unauthorized switch connections
* Protects STP topology integrity
* Automatically disables offending ports

---

## ✅ VLAN Verification

Use the following command before production deployment:

```bash
show vlan brief
```

Verify:

* Every port is assigned to the correct VLAN
* No unused ports remain active
* Trunk ports are properly configured
