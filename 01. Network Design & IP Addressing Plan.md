# Network Design & IP Addressing Plan

> 📘 **Document Type:** Network Design Document (NDD)
> 
> 🏢 **Department:** Information Technology (IT)
> 
> 🎯 **Purpose:** Define a scalable and segmented network topology, establish an IP addressing scheme using subnetting, and document the network architecture before deployment and device configuration.

---

# Objective

Design a scalable, segmented network topology for a site, create an IP addressing scheme using subnetting, and produce a **Network Design Document (NDD)** before any device configuration.

---

# Sample Site Topology

## Network Architecture Overview

| Layer                | Device Role                  | Description                     |
| -------------------- | ---------------------------- | ------------------------------- |
| 🌐 Internet          | ISP Router / Modem           | Provides Public IP connectivity |
| 🔥 Core/Edge         | Firewall (pfSense/FortiGate) | WAN + LAN Gateway               |
| 🧠 Distribution Core | L3 Switch (24-port)          | Inter-VLAN routing              |
| 🖧 Access Layer      | L2 PoE Switches (per floor)  | Connects end-user devices       |
| 📶 Wireless          | Wireless Access Points (APs) | Supports Wi-Fi clients          |
| 📹 CCTV Segment      | Dedicated PoE Switch         | Connects IP Cameras + NVR       |

# Step-by-Step Network Design Process

---

## Step 1: Gather Requirements

📋 Identify and document the following:

* Number of users
* Devices per department
* Bandwidth requirements
* Security zones
* Future growth estimates

---

## Step 2: Choose a Private IP Range

🧮 Select an appropriate private IP addressing scheme:

* Use `192.168.x.x` for small sites
* Use `10.x.x.x` for large enterprise environments
* Use `/24` subnets per VLAN

---

## Step 3: Assign VLAN IDs and Subnets

🗂️ Assign VLAN IDs and subnets according to the VLAN segmentation table above.

### IP Reservation Guidelines

Reserve the first 10 IP addresses in each subnet for infrastructure devices, including:

* Default gateway
* Switches
* Servers
* Access points
* Network appliances

---

## Step 4: Define Default Gateway per VLAN

🌐 Configure the default gateway for each VLAN.

### Standard Practice

Typically assign `.1` as the gateway address.

#### Example

```text
192.168.40.1 → CCTV VLAN Gateway
```

The gateway should be assigned to either:

* Layer 3 Switch
* Firewall

---

## Step 5: Plan DHCP Scope

📦 Define DHCP address allocation ranges.

### DHCP Configuration Standards

| Configuration Item       | Value      |
| ------------------------ | ---------- |
| Excluded Static IP Range | `.1 – .20` |
| DHCP Start Address       | `.21`      |
| DHCP End Address         | `.200`     |
| User Lease Time          | `8 hours`  |
| Device Lease Time        | `24 hours` |

---

## Step 6: Design Physical Connectivity

🔌 Plan the physical network infrastructure and uplinks.

### Connectivity Standards

* Core switch uplinks should use `10G SFP+`
* Access switch uplinks to the core should be:

  * Minimum `1G`
  * Or configured using `LAG (Link Aggregation)`

---

## Step 7: Document Every Device

📝 Maintain detailed records for all network devices.

### Required Documentation Fields

* Hostname
* Management IP
* VLAN Assignment
* Physical Location
* MAC Address (where known)

---

## Step 8: Create Network Diagrams

🖼️ Produce both logical and physical diagrams.

### Recommended Tools

* draw.io
* Microsoft Visio

### Required Diagrams

* Logical Network Diagram
* Physical Rack Diagram

---

## Step 9: Peer Review the Design

👥 Conduct a technical review before implementation.

### Review Objectives

* Validate network segmentation
* Confirm scalability
* Verify IP allocation consistency
* Identify security or redundancy gaps

The design should be reviewed by:

* A colleague
* Senior Network Engineer
* Technical Lead

# Best Practice Tips

## ✅ Network Segmentation

* Always separate CCTV traffic onto its own VLAN.
* This prevents cameras from being accessed by office PCs.
* Reduces broadcast domains and improves security.

---

## 📈 Capacity Planning

* Plan for at least **20% future growth**.
* Example:

  * A site with `80 users` today may grow to `96 users` next year.

---

## 🗃️ IP Address Management (IPAM)

* Document the IP plan in a spreadsheet or IPAM solution.
* Keep documentation updated regularly.
* Undocumented networks become difficult to manage and troubleshoot.

---

## 🌐 Subnetting Recommendation

* Use `/24` subnets for standard VLAN deployments where practical.
