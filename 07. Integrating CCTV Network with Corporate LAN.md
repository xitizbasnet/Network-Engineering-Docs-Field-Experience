# Integrating CCTV Network with Corporate LAN

> 📘 **Document Type:** Network Integration & Security Architecture Guide
> 
> 🏢 **Department:** Information Technology (IT)
> 
> 🎯 **Objective:** Securely integrate the CCTV VLAN (cameras + NVR) with the corporate LAN while enforcing strict isolation and controlled access for authorised security personnel.

---

# Objective

Securely integrate the CCTV VLAN (cameras + NVR) with the corporate LAN, allowing authorised security staff to access the NVR remotely while ensuring cameras remain fully isolated from office traffic.

---

# Integration Architecture

## Network Segmentation Overview

| Segment               | IP Range                         | Role                                      |
| --------------------- | -------------------------------- | ----------------------------------------- |
| 📹 CCTV VLAN 40       | `192.168.40.0/24`                | Cameras + NVR                             |
| 🏢 Staff VLAN 20      | `192.168.20.0/24`                | Office PCs                                |
| 🧠 Management VLAN 10 | `192.168.10.0/24`                | Network devices                           |
| 🔥 Firewall (pfSense) | `192.168.x.1` (per VLAN gateway) | Controls inter-VLAN traffic               |
| 🎥 NVR                | `192.168.40.100`                 | Recording server (restricted access only) |

---

# Step-by-Step Integration Process

---

## Step 1: Verify CCTV VLAN Routing

📡 Ensure:

* VLAN 40 is trunked across all required switches
* VLAN 40 is routed via:

  * L3 switch **or**
  * Firewall

---

## Step 2: Block Staff Access to CCTV VLAN

🔥 Firewall Rule — Staff to CCTV (DENY)

📍 Location:

`Firewall > Rules > VLAN20`

### Rule Configuration:

| Field       | Value             |
| ----------- | ----------------- |
| Source      | `192.168.20.0/24` |
| Destination | `192.168.40.0/24` |
| Action      | ❌ Block           |

🚫 Purpose:

* Prevent office PCs from accessing cameras or CCTV network devices

---

## Step 3: Allow Security Team Access to NVR

🔐 Firewall Rule — NVR Access (ALLOW)

### Rule Configuration:

| Field       | Value                     |
| ----------- | ------------------------- |
| Source      | `Security_Team_PCs_Alias` |
| Destination | `192.168.40.100`          |
| Ports       | `TCP 80, 443, 8000`       |
| Action      | ✅ Allow                   |

📌 Purpose:

* Enables secure remote access to NVR interface for authorised users only

---

## Step 4: Block Cameras from LAN Access

📹 Firewall Rule — CCTV Isolation

### Rule Logic:

* Block all outbound traffic from:

  * `192.168.40.0/24`
* Exception:

  * Access to `192.168.40.100` (NVR only)
  * DNS (name resolution)
  * NTP (time synchronization)

🚫 Purpose:

* Prevent cameras from reaching internal corporate network

---

## Step 5: Allow NVR Time Synchronization

⏱️ Firewall Rule — NTP Access

* Allow NVR to reach:

  * `192.168.10.x` NTP server

📌 Purpose:

* Ensures accurate timestamps for recordings and logs

---

## Step 6: Enable NVR Notifications

📧 Firewall Rule — SMTP / Alerting

Allow NVR to access:

* SMTP server
* Push notification service

📌 Purpose:

* Enables email alerts and security notifications

---

## Step 7: Configure DHCP Reservation for NVR

🧾 Ensure static IP assignment:

| Parameter   | Value                     |
| ----------- | ------------------------- |
| Device      | NVR                       |
| MAC Address | Registered in DHCP server |
| IP Address  | `192.168.40.100`          |

📌 Purpose:

* Ensures consistent and reliable access to NVR

---

## Step 8: Validate Security Team Access

🧪 Test from authorised PC:

```text id="nvr_test_001"
http://192.168.40.100:80
```

✅ Expected Result:

* NVR web interface loads successfully

---

## Step 9: Validate Office Network Restriction

🧪 Test from standard office PC:

* Ping `192.168.40.100`

❌ Expected Result:

* Request times out (blocked by firewall)

---

## Step 10: Verify Camera Isolation

🧪 Test from office PC:

* Ping any camera in `192.168.40.x`

❌ Expected Result:

* No response (fully isolated)

---

## Step 11: Verify NVR Internet Access

🌐 From NVR:

```bash id="nvr_net_test"
ping 8.8.8.8
```

✅ Expected Result:

* Successful response via NAT/firewall

---

## Step 12: Documentation & Handover

📄 Ensure all configurations are documented in:

* Network Design Document (NDD)
* Firewall rule logs
* VLAN configuration sheets

📌 Deliver to client for final approval and handover

# Best Practice Tips

## 🔐 Least Privilege Access

* Only authorised Security Team PCs should access the NVR
* Never expose CCTV systems to entire office subnet

---

## 📊 Firewall Logging

* Enable logging on:

  * Staff → CCTV block rule

📌 Purpose:

* Detect unauthorized access attempts
* Improve incident response visibility

---

## 🏷️ Physical Security

* Label all CCTV PoE switch ports clearly
* Prevent accidental plugging of office devices into CCTV VLAN

---

## 🔁 Access Review Cycle

* Perform quarterly review of:

  * Security Team alias
  * User access permissions

🚫 Immediately remove access for:

* Ex-employees
* Unauthorised devices

---

## 🔐 Secure Remote Access

* Enable HTTPS (SSL/TLS) on NVR
* Disable HTTP where possible

📌 Purpose:

* Encrypt video management traffic
* Prevent credential interception

---

## 📜 Compliance Consideration

This architecture aligns with industry CCTV security standards such as:

* NPSA guidelines
* BS EN 62676 (Video surveillance systems)

📌 Ensure compliance is documented in project handover report
