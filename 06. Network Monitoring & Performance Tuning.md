# Part A — Enable SNMP on Network Devices (Cisco)

## SNMP Configuration

📡 Configure SNMP for network monitoring and telemetry.

```bash id="snmp001"
snmp-server community PUBLIC_READ ro
snmp-server community PRIVATE_WRITE rw
snmp-server host 192.168.50.10 version 2c PUBLIC_READ
snmp-server enable traps
snmp-server location Server-Room-Rack-A
snmp-server contact netadmin@company.com
```

---

## SNMP Configuration Breakdown

| Parameter          | Purpose                             |
| ------------------ | ----------------------------------- |
| `PUBLIC_READ`      | Read-only monitoring access         |
| `PRIVATE_WRITE`    | Write access (restricted/admin use) |
| `snmp-server host` | Defines monitoring server           |
| `traps`            | Sends real-time alerts              |
| `location`         | Physical device location            |
| `contact`          | Administrator contact info          |

# Part C — QoS Configuration (CCTV & VoIP Prioritization)

## QoS Policy Overview

🎯 Purpose:

* Prioritize VoIP traffic
* Prioritize CCTV video streams
* Ensure stable performance under load

---

## Class Map Configuration

```bash id="qos001"
class-map match-any VOIP-TRAFFIC
 match ip dscp ef

class-map match-any CCTV-TRAFFIC
 match ip dscp af31
```

---

## QoS Policy Definition

```bash id="qos002"
policy-map WAN-QOS
 class VOIP-TRAFFIC
  priority 20000

 class CCTV-TRAFFIC
  bandwidth 50000

 class class-default
  fair-queue
```

---

## Apply QoS to Interface

```bash id="qos003"
interface GigabitEthernet0/0
 service-policy output WAN-QOS
```

---

## QoS Configuration Explanation

| Class        | Purpose         | Allocation         |
| ------------ | --------------- | ------------------ |
| VOIP-TRAFFIC | Voice calls     | 20 Mbps guaranteed |
| CCTV-TRAFFIC | Video streaming | 50 Mbps bandwidth  |
| DEFAULT      | General traffic | Shared fair queue  |

# Key Metrics to Monitor

## Network Health Indicators

| Metric                | Location          | Threshold                            |
| --------------------- | ----------------- | ------------------------------------ |
| CPU / Memory          | Switches, Routers | >80% warning, >95% critical          |
| Interface Utilisation | Uplinks           | >70% sustained = congestion risk     |
| Packet Loss           | WAN / Core links  | >0.1% requires investigation         |
| Latency               | WAN / Inter-site  | >20ms LAN abnormal; >150ms WAN issue |
| PoE Power Usage       | PoE Switches      | >85% risk of device failure          |
| Disk Usage            | NVR / Servers     | >85% triggers cleanup alert          |

# Best Practice Tips

## 🔐 SNMP Security

* Use **SNMPv3 in production environments**
* SNMPv3 provides:

  * Authentication
  * Encryption
  * Secure monitoring

🚫 Avoid SNMPv1/v2c in production due to lack of security.

---

## 📊 Alert Strategy

* Set alerts BEFORE critical thresholds:

  * Example: 80% CPU = warning, not failure
* Enables proactive troubleshooting

---

## ⚡ PoE Power Monitoring

* Always monitor PoE budget alongside camera counts
* Adding devices can silently overload switches

---

## 📈 Network Baseline

* Establish performance baseline during normal operation
* Enables anomaly detection:

  * Traffic spikes
  * Latency changes
  * Bandwidth misuse

---

## 💾 Configuration Backup Automation

* Use tools such as:

  * RANCID
  * Oxidized

⚠️ Purpose:

* Prevent configuration loss
* Enable fast recovery after hardware failure
