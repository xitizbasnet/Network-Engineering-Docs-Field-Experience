# Objective

Configure a network firewall (pfSense / FortiGate / Cisco ASA) with zone-based policies, inbound and outbound rules, CCTV remote access rules, and IPS/IDS settings.

---

# Firewall Zones & Default Policy

## Security Zone Architecture

| Zone               | Segment                 | Default Policy                                          |
| ------------------ | ----------------------- | ------------------------------------------------------- |
| 🔴 WAN (Untrusted) | Internet (ISP)          | Block ALL inbound by default                            |
| 🟢 LAN (Trusted)   | Staff VLAN 20, Servers  | Allow internet access; restrict inter-VLAN traffic      |
| 📹 CCTV Zone       | VLAN 40 (Cameras + NVR) | Block access to LAN; allow controlled NVR remote access |
| 🟡 DMZ             | Public-facing servers   | Limited inbound from WAN; fully isolated from LAN       |
| 📶 Guest Zone      | VLAN 30 (Guest Wi-Fi)   | Internet-only access; block all internal network access |

# Step-by-Step pfSense Firewall Configuration

## Step 1: Access Web Interface

🌐 Open pfSense Web UI:

```
https://192.168.10.1
```

### Default Login (Change Immediately)

| Field    | Value     |
| -------- | --------- |
| Username | `admin`   |
| Password | `pfsense` |

⚠️ **Security Requirement:** Change default credentials immediately after first login.

---

## Step 2: System Configuration

Navigate to:

📍 `System > General Setup`

Configure:

* Hostname
* Domain name
* DNS Servers:

  * `8.8.8.8`
  * `1.1.1.1`
* Timezone:

  * `Asia/Kolkata`

---

## Step 3: Interface Assignment

Navigate to:

📍 `Interfaces > Assignments`

Configure:

* WAN → `em0`
* LAN → `em1`

➕ Add VLAN interfaces for:

* CCTV
* Guest Network

---

## Step 4: VLAN Interface Creation

Navigate to:

📍 `Interfaces > VLANs`

Create VLANs:

| VLAN ID | Purpose |
| ------- | ------- |
| VLAN 40 | CCTV    |
| VLAN 30 | Guest   |

Attach VLANs to the LAN parent interface.

---

## Step 5: WAN Firewall Rule (NVR Access)

Navigate to:

📍 `Firewall > Rules > WAN`

### Add Rule:

* Protocol: TCP
* Destination: NVR IP
* Port: `8000`
* Action: Allow (specific rule only)

🚫 **Important:** Block all other inbound traffic by default.

---

## Step 6: CCTV Zone Rules

Navigate to:

📍 `Firewall > Rules > CCTV`

### Rules Configuration:

* Allow CCTV → NVR communication (same VLAN)
* Block CCTV → LAN access
* Allow NVR → Internet (for P2P/DDNS services)

---

## Step 7: Guest Network Rules

Navigate to:

📍 `Firewall > Rules > Guest`

### Rules Configuration:

* Allow Guest → WAN:

  * Ports: `80`, `443`
* Block Guest → ALL internal VLANs

📶 Result:

* Internet access only
* No internal network visibility

---

## Step 8: LAN Firewall Rules

Navigate to:

📍 `Firewall > Rules > LAN`

### Rules Configuration:

* Allow LAN → Internet access
* Define controlled inter-VLAN rules:

  * Example: Staff → Server VLAN access (restricted)

---

## Step 9: Enable IDS/IPS

Navigate to:

📍 `Services > Snort` or `Suricata`

### Configuration:

* Enable on WAN interface
* Enable **Emerging Threats (ET) ruleset**

🛡️ Purpose:

* Detect malicious traffic
* Prevent intrusion attempts
* Monitor external threats

---

## Step 10: Enable pfBlockerNG (Optional but Recommended)

📍 DNS-level protection:

* Ad blocking
* Malware domain filtering
* Threat intelligence-based blocking

---

## Step 11: Rule Testing & Validation

🧪 Perform validation tests:

### Test Cases:

* Attempt ping from Staff VLAN → CCTV VLAN
  ❌ Should be blocked

* Verify NVR remote access
  ✅ Should work as configured

---

## Step 12: Firewall Aliases

Navigate to:

📍 `Firewall > Aliases`

### Create Alias Groups:

Examples:

* `CCTV_Devices`
* `Admin_PCs`

### Benefits:

* Simplifies rule management
* Improves readability
* Reduces configuration errors

---

# Important Warnings

## 🚨 WAN Security

* Default policy must be **deny all inbound traffic**
* Only explicitly required ports should be opened

⚠️ Every open port increases attack surface.

---

## 📹 CCTV Security Risk

* Never expose NVR or IP cameras directly to the public internet
* IP cameras are frequently targeted by:

  * Botnets
  * Automated scanners
  * Credential attacks

---

## 📶 Guest Network Isolation

* Guest Wi-Fi must be fully isolated from internal networks
* Prevents lateral movement attacks
* Stops compromised devices from accessing internal systems

---

## 🔄 Firmware Updates

* Keep firewall firmware updated regularly
* Critical security patches are frequently released

---

# Best Practice Tips

## 🗂️ Use Firewall Aliases

* Group IPs and ports logically
* Improves rule clarity and scalability
* Reduces configuration complexity

---

## 📊 Enable Logging

* Enable logging for all **deny rules**
* Provides critical forensic data for:

  * Security investigations
  * Incident response
  * Traffic analysis

---

## 💾 Backup Strategy

* Schedule automatic firewall configuration backups
* Store backups in a secure location

⚠️ Firewall recovery without backups can be time-consuming and complex.

---

## 🧪 Testing Best Practice

* Always test firewall rules from an **end-user device**
* Avoid testing only from firewall itself

Reason:

* Local traffic may bypass certain filtering rules
* Real-world validation ensures accuracy
