# Objective

Configure a router for WAN connectivity, static and dynamic routing (OSPF), NAT/PAT for internet access, and inter-VLAN routing on an L3 switch.

---

# Part A — Router Initial Configuration & WAN Setup

## Step 1: Console Access

🔌 Connect to the router using a console cable.

```bash id="rtr001"
enable
configure terminal
```

---

## Step 2: Basic System Configuration

🖥️ Configure hostname and disable DNS lookup.

```bash id="rtr002"
hostname ROUTER-01
no ip domain-lookup
```

### Purpose

| Configuration         | Purpose                                 |
| --------------------- | --------------------------------------- |
| `hostname ROUTER-01`  | Sets device identity                    |
| `no ip domain-lookup` | Prevents DNS delay on mistyped commands |

---

## Step 3: Configure WAN Interface (ISP Uplink)

🌐 Configure the interface connected to the ISP.

```bash id="rtr003"
interface GigabitEthernet0/0
 ip address <ISP_ASSIGNED_IP> <SUBNET_MASK>
 no shutdown
```

---

## Step 4: Configure LAN Interface (Core Switch Connection)

🖧 Configure internal LAN gateway interface.

```bash id="rtr004"
interface GigabitEthernet0/1
 ip address 192.168.10.1 255.255.255.0
 no shutdown
```

---

## Step 5: Configure Default Route

📡 Define default route toward ISP gateway.

```bash id="rtr005"
ip route 0.0.0.0 0.0.0.0 <ISP_GATEWAY_IP>
```

---

## Step 6: Verify Routing Table

🔍 Confirm default route installation.

```bash id="rtr006"
show ip route
```

### Expected Output

* `S* 0.0.0.0/0` → indicates static default route is active

---

# Part B — NAT/PAT Configuration

## Overview

🌍 NAT (Network Address Translation) enables internal private IP addresses to access the internet using a single public IP address.

---

## NAT Configuration

```bash id="nat001"
access-list 1 permit 192.168.0.0 0.0.255.255
```

---

## Define NAT Interfaces

### WAN Interface (Outside)

```bash id="nat002"
interface GigabitEthernet0/0 
 ip nat outside
```

### LAN Interface (Inside)

```bash id="nat003"
interface GigabitEthernet0/1
 ip nat inside
```

---

## Enable PAT (Overload NAT)

🔁 Allows multiple internal hosts to share one public IP.

```bash id="nat004"
ip nat inside source list 1 interface GigabitEthernet0/0 overload
```

---

# Part C — Inter-VLAN Routing (L3 Switch)

## Overview

🔀 Enables communication between VLANs using Layer 3 switching (alternative to router-on-a-stick).

---

## Enable Routing on L3 Switch

```bash id="l3r001"
ip routing
```

---

## Configure VLAN Interfaces (SVIs)

### Staff VLAN Gateway

```bash id="l3r002"
interface Vlan20
 ip address 192.168.20.1 255.255.255.0
 no shutdown
```

---

### CCTV VLAN Gateway

```bash id="l3r003"
interface Vlan40
 ip address 192.168.40.1 255.255.255.0
 no shutdown
```

# Part D — OSPF Configuration (Multi-Site Routing)

## Overview

🌐 OSPF (Open Shortest Path First) enables dynamic routing between multiple network sites.

---

## OSPF Configuration

```bash id="ospf001"
router ospf 1
 router-id 1.1.1.1
 network 192.168.10.0 0.0.0.255 area 0
 network 192.168.20.0 0.0.0.255 area 0
 network 192.168.40.0 0.0.0.255 area 0
 passive-interface GigabitEthernet0/0
```

---

## OSPF Configuration Notes

| Setting             | Purpose                        |
| ------------------- | ------------------------------ |
| `router-id`         | Manually defines OSPF identity |
| `network`           | Advertises connected networks  |
| `area 0`            | Backbone OSPF area             |
| `passive-interface` | Prevents OSPF updates on WAN   |

# Best Practice Tips

## 🔑 OSPF Stability

* Always manually configure `router-id`
* Prevents instability caused by interface changes
* Ensures consistent OSPF adjacency

---

## 🔇 Passive Interfaces

* Mark all non-routing interfaces as passive
* Especially:

  * ISP-facing interfaces
  * Access ports

This prevents:

* Unwanted OSPF hello packets
* Security exposure
* Routing table pollution

---

## 🌍 NAT Configuration Safety

* Always use ACL-based NAT rules
* Example:

```bash id="nat005"
ip nat inside source list <ACL> interface <WAN_INTERFACE> overload
```

🚫 Avoid:

* NATing all traffic without filtering

---

## 🧪 Testing Strategy

* Always test inter-VLAN routing from an **end-user device**
* Avoid testing only from router or switch CLI
* This ensures real-world traffic validation

---

## 💾 Configuration Management

* Save configuration after every successful change:

```bash id="save001"
write memory
```

OR

```bash id="save002"
copy running-config startup-config
```
